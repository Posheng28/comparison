# 注意細節條件面板 + 款一~六 引擎重構 設計文件

**日期**：2026-05-30
**目標**：把處置工具的「近 6 日收盤價表格」換成 attstock 風格的「注意細節條件」面板（每一款都有條件總結），並把條文引擎範圍收斂到款一~六、加上款四/五、移除款十一/十二；款三/四（及款六項三）的「成交量/週轉率」第二條件升級成可算的硬量門檻，盤中即時更新。
**架構**：條件引擎（`lib/clauseEngine.ts`，純函式、可單測）產出「足以渲染 attstock 風格逐項細節」的結構化結果；UI 層（`components/DisposalTool.tsx`）以 6 張可收合卡片呈現，錨定「計算日」，盤中帶入 Yahoo 即時價與累積量。
**技術棧**：Next.js 16.2.6（Turbopack、App Router）、TypeScript strict、React 19、Tailwind 4、vitest 4。`npm run build` 為 CI gate。

> ⚠️ Next.js 版本有破壞性改動，動到 framework API 前先讀 `node_modules/next/dist/docs/`。

## 1. 背景與範圍

### 1.1 為什麼做
- 現況：處置面板上方是「近 6 日收盤價」唯讀表（5~6 欄逐日門檻），資訊密度低、看不出「為何/哪一款會觸發」。
- attstock（attstock.tw，本專案權威對拍工具）對每一款都列出「條件總結」逐項拆解（法規門檻 → 各 AND 子條件 → 現值 → 燈號），一眼看懂。使用者要同款體驗，放在原本 6 個價格的位置，標題「注意細節條件」。

### 1.2 範圍（只做款一~六）
- **保留並重構**：款一①、款一②、款二、款三、款四（新增）、款五（新增）、款六。
- **移除**：款十一（起迄價差級距）、款十二（借券賣出）。
  - 法源：FL007225 第 6 條第 1 項第 2 款——自動處置「連 N 日」只計**第四條第一項第一款至第八款**；款九~十四（含本工具曾用的款十一/十二）不計入。移除不影響自動處置預測（僅「純款七日」可能少算，罕見；款七需融資券資料、不可測，本輪不做）。
- **錨定**：面板錨定「計算日」（＝下一個待判定交易日）。到計算日盤中時，仍判定該計算日，並帶入 Yahoo 即時價＋即時累積成交量計算。

## 2. 法規依據（已核對原文，TWSE FL007226，2024-12-31 版）

款三/四/五 **第一條件完全相同**＝款一②前半＝引擎現有 `t3` 觸發價：
> 最近 6 個營業日（含當日）累積收盤價漲跌% > 25%，且漲跌%與**全體**及**同類**有價證券平均值之**差幅均 ≥ 20%**。

各款第二條件（上市值）：

| 款 | 名稱 | 第一條件（價格） | 第二條件 | 除外重點 |
|---|---|---|---|---|
| 一① | 累積漲跌幅異常 | 累積 **>32%** 且差幅≥20% | — | 見 §5.4 七項 |
| 一② | 累積漲跌幅異常 | 累積 **>25%** 且差幅≥20% **且**起迄兩日價差 ≥ **50 元** | — | 同上 |
| 二 | 中長期漲跌異常 | 30日>**100%**(差幅≥85%)／60日>**130%**(≥110%)／90日>**160%**(≥135%) | 且收盤須高於(漲)/低於(跌)當日開盤參考價 | 見第 3 條八項 |
| 三 | 漲跌異常+量能放大 | 累積>25%+差幅≥20% | 當日量 ≥ 近 **60** 營業日日均量 **×5 倍**，且放大倍數與全體均值差 ≥ **4 倍** | 週轉率<0.1% 或 量<500 張不適用 |
| 四 | 漲跌異常+高週轉 | 累積>25%+差幅≥20% | 當日**週轉率 ≥ 10%**，且與全體均值差幅 ≥ **5%** | 同類<5種/PE負或≥60倍不適用類股 |
| 五 | 漲跌異常+券商集中 | 累積>25%+差幅≥20% | 券商受託買賣**集中度 > 25%**（每分支+1%，上限35%）且 > **500 張** | 同上 |
| 六 | 本益比/股淨比異常 | （非價格門檻，見 §4） | 四項同時成立，見 §4 | IPO不計／**非普通股不適用**／鉅額交易扣除 |

## 3. 款三/四/五：成交量門檻設計（核心）

### 3.1 關鍵恆等式
> **當日週轉率 = 當日成交股數 ÷ 發行股數**

因此第二條件可反推成硬「成交量（張）」門檻，盤中量一路累積即時比對：

- **款四**：週轉率 ≥ 10% ⟺ 成交股數 ≥ 0.1×發行股數 ⟺ **成交量(張) ≥ 0.1 × 發行張數**。
  - 例：發行 2,071,465 張 → 門檻 207,147 張（對上 attstock 截圖）。
  - ⚠️ **上市門檻 ≥10%；上櫃 >5%** → 上櫃量門檻 = 0.05 × 發行張數。
- **款三**：**成交量(張) ≥ 5 × 近 60 營業日日均量(張)**（上市/上櫃同為 5 倍）。
- **款五**：券商集中度為非公開券商資料 → 維持「假設達標」開關（無法事前算）。

### 3.2 殘差（市場比較項）一律「假設成立」
- 款三「放大倍數與全體均值差 ≥ 4 倍」、款四「週轉率與全體均值差 ≥ 5%」需要掃全市場量/週轉率均值（重，且發行股數需全市場）。
- 實證：全市場平均週轉率約 1.49%，+5% = 6.49% < 絕對門檻 10% → **10% 一定是綁定門檻**（「週轉率≥10% 就過第二條」）。故全市場量/週轉率均值**跳過不算**，殘差以「次要條件假設成立」開關帶過（預設**開**）。
- 每款一個開關：`c3Assume`（款三+4倍）、`c4Assume`（款四+5%）、`c5Assume`（款五整個第二條件）。

### 3.3 三款獨立呈現（使用者選定）
三條各一行、價格欄都＝t3（相同），第二條件各自顯示量門檻/集中度。**不合併**。

## 4. 款六：四項「同時(AND)」成立 + 修正

法規（第 7 條）寫「當日**同時達下列各款**情事者」→ 四項全部 AND：

| 項 | 條件 | 處理 |
|---|---|---|
| 一 PE | PE<0 **或**（PE≥60 倍 且 > 全體均值×2） | 可算 |
| 二 PBR | PBR≥6 倍 **且** > 全體均值×2 | 可算 |
| 三 週轉率+量 | 當日週轉率 ≥ 5% **且** 量 ≥ 3000 張 | **可算**（有發行股數）→ 量 ≥ max(0.05×發行張數, 3000 張) |
| 四 三選一 | (1)PBR>產業均×4 (2)券商金額集中≥10%且≥1億 (3)投資人集中≥10%且≥1億 | 多為非公開/缺產業PBR → **假設達標**(`c6Assume`) |

除外 3 項：①IPO 無漲跌幅期間不計 ②**非普通股不適用**（本工具只看普通股 `[1-9]\d{3}`→恆適用）③鉅額交易在算項三/項四(2)(3)時扣除成交部分。

> **上櫃款六差異**（LW10819047）：PE ≥65 倍／PBR ≥4 倍（上市 60/6）；項三量門檻 ≥2000 張（上市 3000）；項四(1) 產業 PBR ×2（上市 ×4）；實收資本<8000 萬時成交金額門檻減半。

**款六觸發 = 項一 ∧ 項二 ∧ 項三 ∧ c6Assume(項四)。**

### 4.1 需修正的既有行為
1. **OR → AND**：現有 `c6` 用 `peHit || pbrHit`（OR），法規是項一 ∧ 項二（PE 與 PBR 須同時異常）→ 改 `&&`。會讓款六更難觸發、更貼法規（多數股維持「無風險」，與 attstock 國巨款六=無風險一致）。
2. **項三升級**：有了發行股數，「週轉率≥5% 且 量≥3000張」變成可算硬門檻，不再丟進假設；款六的 `c6Assume` 只剩**項四**。
3. **PE/PBR 全體均值**沿用既有**中位數**（非法規的「發行單位數加權平均」）——避免高 PE 成長股灌爆 ×2 閘門，先前已驗證的決定，程式註解標明法源差異。

> 對拍 attstock 款六驗證 AND 解讀；若 attstock 實測較寬鬆，以對拍為準回頭調整（見 §10）。

## 5.「注意細節條件」面板（取代近 6 日表格）

### 5.1 整體
- 位置：取代現有「近 6 日收盤價」唯讀表（DisposalTool 內，約 869–917 行區塊）。可編輯的價格卡保留。
- 標題：「注意細節條件」，副標顯示計算日與資料狀態（如「計算日 6/02・盤中即時 14:01」或「預估」）。
- 內容：**6 張可收合卡片**（款一~六），收合顯示門檻總結＋燈號，展開顯示逐項條件。

### 5.2 卡片（收合）— 比照 attstock 截圖
```
第N款 〈名稱〉   〈門檻總結〉                       [燈號] ▾
```
門檻總結範例：
- 款一：`收盤≥742`
- 款二：`收盤≥788(+6.78%)`
- 款三：`收盤≥800(+8.40%) 且 量≥244,367張`
- 款四：`收盤≥800(+8.40%) 且 量≥207,147張`
- 款五：`收盤≥800(+8.40%)（且券商佔比>25%）`
- 款六：`不會觸發`（或 PE/PBR 概況）

燈號（比照 attstock）：**無風險**（綠，價格不可能達/條件不成立）／**可能觸發**（橘，盤中或預估可能達）／**已觸發**（紅，現價已達門檻）。

### 5.3 卡片（展開）— 逐項條件總結
共同結構：
1. **法規行**：一句話法規條件 + `[法規]` 標籤。例：款四「6日累積漲跌 ≥25% 且差幅≥20% 且週轉率 ≥10% 且差幅≥5%」。
2. **共用區塊**（款一/三/四/五/六 適用）：`已知累積 +X%(N日) │ 全體均值 +a% 差幅 d1(需≥20%) │ 同類均值 +b% 差幅 d2`。
3. **群組逐列**：每個 AND 子條件一組，標題顯示該群門檻與燈號，底下列現值/明細：
   - 款三/四/五的「漲跌幅」群：`漲跌幅 ≥ effective%（門檻25%→25.14% 拉高門檻）`→ 子列「漲跌幅 收盤≥t3」「差幅 門檻25%→25.14%（需超出均值20%以上）」。
   - 款四「週轉率」群：`週轉率 ≥ 10%（≈207,147張）`→ 子列「基本門檻 ≥10%」「差幅條件（與全體差≥5%）次要假設成立」「參考：發行張數 W」。盤中再列「目前 V 張 / 門檻 Y 張（週轉率 p% / 10%）」。
   - 款三「量能」群：`量 ≥ 5×近60日均量 = X張`→ 子列「基本門檻 5 倍」「放大倍數與全體差≥4倍 次要假設成立」「參考：近60日均量 Z 張」「目前 V 張 / 門檻 X 張」。
   - 款五「券商集中」群：`集中度 > 25%（非公開）`→「假設達標」開關。
   - 款六四項群：項一PE／項二PBR／項三週轉率+量／項四三選一（假設達標）。
4. **除外條件**（可收合）：完整列出，標示 ✓可判（如 收盤<5元、同類<5種、PE負或≥60倍）與 —未實作（IPO 無漲跌幅、除權息調整、溢/折價、認購售權證）。

### 5.4 款一除外 7 項（FL007226 第 2 條第 2 項，與 attstock 一致）
1. IPO 無漲跌幅期間 — 未實作（標示）
2. 除權息等非交易因素 — 未實作
3. 收盤 < 5 元不適用 — ✓可判
4. 同類 < 5 種不適用類股規定 — ✓可判（sectoravg 回傳 n）
5. PE 負或 ≥ 60 倍不適用類股規定 — ✓可判（有 PE）
6. 前一營業日溢/折價 ≤10% 或反向 — 未實作
7. 認購售權證特例 — 未實作（普通股 N/A）

## 6. 盤中即時（價 + 量）

- 計算日 = 下一個待判定交易日。盤中（計算日當天、台灣時間 13:30 收盤前/~14:00 定案前）：Yahoo `chart` 最後一根為**即時未定案**，其 `close` = 即時價、`volume` = 當日累積成交量。
- **價格判定**：沿用現有「收盤判定 <14:00 丟今天那根」邏輯避免把未定案當收盤；但「正在判定的計算日」本身就是用即時價跑各款門檻。
- **量門檻判定**：款三/四（及款六項三）的量比對**使用即時累積量**（不丟），面板顯示「目前 V 張 / 門檻 X 張」並隨盤中更新。
- 收盤定案後（~14:00），計算日成為已完成日，計算日往後滾動到下一交易日。

## 8. `lib/clauseEngine.ts` 重構

### 8.1 型別
```ts
type ClauseId = '1①' | '1②' | '2' | '3' | '4' | '5' | '6'   // 移除 '11' '12'

type CondStatus = 'met' | 'possible' | 'safe' | 'assumed' | 'raised'  // 已達/可能/無風險/假設/拉高門檻
interface SubCond { label: string; threshold: string; current?: string; status: CondStatus; note?: string }
interface CondGroup { title: string; threshold: string; status: CondStatus; subs: SubCond[] }

// ClauseResult 擴充：保留 fired/first（給 summarize 計數），加上面板渲染所需結構
interface ClauseResult {
  id: ClauseId
  name: string                  // '漲跌異常+高週轉'
  lawText: string               // 法規行
  fired: boolean
  first: boolean                // 是否「第一款」性質（給規則①連3日）
  badge: 'safe' | 'possible' | 'fired'
  headerThreshold: string       // 收合門檻總結
  groups: CondGroup[]           // 展開逐項
  exclusions?: { label: string; status: 'met' | 'unimpl' | 'na' }[]
  blocked?: boolean             // 價格達標但次要假設未開
}
```
> `summarize(rs)` 僅讀 `fired`/`first`，介面相容；面板讀其餘欄位。單一評估、單一真相來源。
